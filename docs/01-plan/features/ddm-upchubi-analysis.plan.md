## 3. 범위 (Scope)

### 3.1 In Scope
1. **파싱 파이프라인**: 42개 `.md` → 통합 JSON/CSV
   - 행 단위 추출, 컬럼 매핑, 데이터 타입 변환
   - LaTeX 노이즈 정제: `$2022 - 07 - 01$` → `2022-07-01`, `$\overline{4}$` → `4`
   - 결측/노이즈 행(`!!!!!!...`) 격리 + 별도 리뷰 리스트
2. **정제·정규화**
   - 일자 표준화(`YYYY-MM-DD`)
   - 금액 숫자화(콤마 제거)
   - 사용자 직책 정규화(예: `의사담당`/`의사팀장`/`의사주임`을 카테고리 매핑)
   - 장소 정규화(공백/특수문자 제거 후 fuzzy 그룹핑)
3. **집계 분석**
   - 시계열: 월별·연도별 총 집행액·건수
   - 사용자(직책)별 TOP N (원본 표기 그대로)
   - 장소별 TOP N + 반복 사용 장소
   - **식당(업소)별 금액 합계 집계** — 총액·건수·평균·최초/최근 사용일·주 사용자 직책 TOP 3
   - 비목·방법(카드/제로페이/법인카드) 분포
   - 인원·금액 분포(1인당 단가)
4. **이상 패턴 탐지 (분포 기반 동적 산정)**
   - 1인당 단가: 분위수(P95/P99) 또는 z-score 기반 outlier 자동 산정
   - 동일 일자 다중 집행 (동일 사용자/직책 기준)
   - 동일 장소 단기 반복 (rolling window 기반 이상 빈도)
   - 휴일·심야(22시 이후) 집행
   - **식당 단위 누적 사용액 outlier**
5. **시각화 (신규 페이지 `ddm-upchubi.html`)**
   - 시계열 라인/바 차트, 도넛(비목/방법), 히트맵(요일×시간대), TOP 리스트
   - **식당 TOP N 차트 + 식당 상세 모달** (시계열·사용자 분포·평균 단가)
   - 데이터 한계·출처·누락 가능성 고지 박스
   - dongdaemun.html과 톤·스타일 일관, 양방향 링크

### 3.2 Out of Scope (이번 cycle)
- 타 자치구 비교 (Phase 2)
- 개별 사용자 신원 식별
- PDF 재OCR (현재 마크다운 입력 그대로 사용)
- 실시간 자동 수집 파이프라인
